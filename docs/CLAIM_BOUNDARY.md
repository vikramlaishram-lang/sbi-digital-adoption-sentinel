# Claim Boundary

This MVP does not execute banking actions.

This MVP does not connect to real SBI systems.

This MVP does not claim access to SBI internal SOPs.

This MVP uses configurable demo SOP/rule logic.

This MVP does not make sensitive eligibility, risk, approval, or limit decisions using an LLM.

The model interprets and explains. The rule engine decides. The receipt records.

The MVP is a readiness and recovery layer for digital adoption, not a production banking decision system.
